# 校园二手交易平台 - User Story

> 本文档按照敏捷开发规范编写，所有User Story按优先级分层，可直接用于Sprint规划

---

## 优先级说明

| 优先级 | 说明 | 迭代计划 |
|--------|------|----------|
| **P0** | 必做 - MVP核心功能，没有它产品无法使用 | Sprint 1-2 |
| **P1** | 重要 - 提升用户体验和平台价值 | Sprint 3-4 |
| **P2** | 可选 - 增值功能，可在后续迭代中添加 | Sprint 5+ |

---

## P0 - 必做（MVP核心功能）

### US-P0-001：用户注册与登录

**用户故事：**
> As a 新用户  
> I want 通过手机号快速注册和登录  
> So that 我可以开始使用平台功能

**优先级：** P0  
**故事点：** 3  
**负责人：** 后端/前端

**验收标准（AC）：**
- [ ] 用户输入手机号可获取短信验证码
- [ ] 验证码5分钟内有效，错误3次后需重新获取
- [ ] 支持手机号+验证码一键注册登录
- [ ] 登录状态保持7天，支持Token刷新
- [ ] 未登录用户访问受限页面时自动跳转登录页

**技术备注：**
- 接入短信服务商（阿里云/腾讯云）
- JWT Token机制，Redis存储会话
- 手机号格式校验：1[3-9]\d{9}

### US-P0-002：校园身份认证

**用户故事：**
> As a 平台运营方  
> I want 用户完成校园身份认证后才能发布商品  
> So that 确保平台用户的真实性和交易安全

**优先级：** P0  
**故事点：** 5  
**负责人：** 后端/前端

**验收标准（AC）：**
- [ ] 用户可提交学工号/教职工号+姓名+学生证照片
- [ ] 提交后状态变为"认证中"，等待审核
- [ ] 运营后台可查看认证申请，支持通过/拒绝
- [ ] 认证通过后方可发布商品，认证状态显示在个人信息页
- [ ] 认证材料仅管理员可见，保护用户隐私

**技术备注：**
- 认证材料存储到OSS/对象存储
- 人工审核流程，后期可对接学校身份认证API

---

### US-P0-003：发布二手商品

**用户故事：**
> As a 学生卖家  
> I want 快速发布我的闲置物品  
> So that 让其他用户看到并购买

**优先级：** P0  
**故事点：** 5  
**负责人：** 后端/前端

**验收标准（AC）：**
- [ ] 支持上传1-9张商品图片，单张≤5MB
- [ ] 必填字段：标题（≤100字）、分类、价格、描述
- [ ] 选填字段：原价、新旧程度（1-6级）、取货方式、取货地址
- [ ] 发布成功后进入"待审核"状态，审核通过后展示
- [ ] 支持保存草稿，草稿仅自己可见

**技术备注：**
- 图片压缩+CDN加速
- 敏感词过滤（商品标题和描述）

**优先级：** P0

### US-P0-006：担保交易下单

**用户故事：**
> As a 学生买家  
> I want 使用担保交易购买商品  
> So that 我的资金安全有保障

**优先级：** P0  
**故事点：** 8  
**负责人：** 后端/前端

**验收标准（AC）：**
- [ ] 买家点击"立即购买"，确认订单信息
- [ ] 买家支付后，资金进入平台冻结账户
- [ ] 卖家收到订单通知，确认发货/约定取货
- [ ] 买家确认收货后，资金释放到卖家账户
- [ ] 7天未确认自动确认收货
- [ ] 支持取消订单（未支付/已支付未发货）

**技术备注：**
- 对接微信支付/支付宝
- 订单状态机：待支付→已支付→已发货→已完成
- 资金流水记录

## P1 - 重要（体验优化）

### US-P1-001：商品收藏

**用户故事：**
> As a 学生买家  
> I want 收藏感兴趣的商品  
> So that 方便后续查看和比较

**优先级：** P1  
**故事点：** 3  
**负责人：** 后端/前端

**验收标准（AC）：**
- [ ] 商品详情页可点击收藏/取消收藏
- [ ] 收藏列表展示商品主图、标题、价格
- [ ] 收藏商品下架时自动移除并通知
- [ ] 收藏商品降价时推送通知（可选）

---

### US-P1-002：交易评价

**用户故事：**
> As a 交易双方  
> I want 在交易完成后互相评价  
> So that 建立信用体系

**优先级：** P1  
**故事点：** 5  
**负责人：** 后端/前端

**验收标准（AC）：**
- [ ] 交易完成后7天内可评价
- [ ] 评价维度：描述相符、交易速度、服务态度（1-5星）
- [ ] 支持文字评价（≤200字）
- [ ] 评价内容公开显示在个人主页
- [ ] 累计好评率计算信用等级

---

### US-P1-003：价格参考

**用户故事：**
> As a 学生卖家  
> I want 获得商品定价参考  
> So that 设置合理的出售价格

**优先级：** P1  
**故事点：** 5  
**负责人：** 后端

**验收标准（AC）：**
- [ ] 发布商品时显示同类商品均价
- [ ] 价格参考基于近30天成交数据
- [ ] 展示价格区间（最低价-最高价）
- [ ] 根据新旧程度给出定价建议

## Sprint 规划建议

### Sprint 1（第1-2周）：用户与认证

- US-P0-003：发布二手商品

**目标：** 完成商品发布和浏览，用户可发布和查看商品

- US-P0-006：担保交易下单

**目标：** 完成核心交易流程，支持担保交易

### Sprint 4（第7-8周）：沟通与管理
- US-P0-008：即时通讯
- US-P0-010：后台商品审核

**目标：** 完成沟通和管理功能，MVP可上线
