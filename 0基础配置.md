> 面向从未搭建过本项目的同学，按顺序完成即可在本地运行。  
> 系统：Windows 10/11（macOS / Linux 步骤类似，命令略有不同）

---

| 服务 | 地址 | 作用 |

## 二、需要安装的软件

### 2.1 Node.js（必装）

**作用：** 运行后端服务、安装前端依赖、启动开发服务器。

1. 打开官网：https://nodejs.org/
2. 下载 **LTS（长期支持版）**，建议 18 或 20 以上
3. 安装时勾选 **Add to PATH**
4. 安装完成后，打开 **PowerShell** 或 **命令提示符**，验证：

```powershell
node -v
npm -v
```

应分别显示版本号，例如 `v20.x.x` 和 `10.x.x`。

---

### 2.2 MySQL（必装）

**作用：** 存储用户、商品、订单、消息等数据。

**方式 A：MySQL 官方安装包（推荐新手）**

1. 下载：https://dev.mysql.com/downloads/mysql/
2. 选择 Windows 安装包，安装类型选 **Developer Default** 或 **Server only**
3. 设置 **root 密码**（务必记住，后面要填进项目）
4. 端口保持默认 **3306**
5. 安装完成后，MySQL 服务应自动启动

**方式 B：小皮/phpStudy、XAMPP 等集成环境**

- 同样需确保 MySQL 已启动，并记住 root 密码

**验证 MySQL 是否运行：**

```powershell
mysql -u root -p
```

输入密码后能进入 `mysql>` 提示符即表示成功。输入 `exit` 退出。

> 若提示 `mysql 不是内部或外部命令`，说明 MySQL 未加入 PATH。可用 **MySQL Workbench** 或 **Navicat** 图形工具代替命令行。

---

### 2.3 Git（选装）

**作用：** 从 GitHub 克隆项目代码。

- 下载：https://git-scm.com/download/win  
- 若已有项目文件夹（如从压缩包解压），可跳过

## 五、配置数据库

### 5.1 创建数据库

用 MySQL 命令行、Workbench 或 Navicat 执行：

```sql
CREATE DATABASE IF NOT EXISTS `2_shou`
  DEFAULT CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;
```

### 5.2 导入基础表结构

**命令行方式（在项目根目录 PBL 下）：**

```powershell
mysql -u root -p 2_shou < 2_shou.sql
```

**图形工具方式：**

1. 连接到 MySQL，选中数据库 `2_shou`
2. 运行 / 导入 `2_shou.sql` 文件

导入后应存在表：`users`、`goods`、`messages`。

### 5.3 扩展表（购物车、订单）

**无需手动导入。** 后端启动时会自动读取 `migrate.sql` 创建：

- `cart_items`
- `orders`
- `order_items`
- `addresses`

若自动迁移失败，可手动执行：

```powershell
mysql -u root -p 2_shou < migrate.sql
```

### 5.4 插入测试用户（可选）

```sql
USE `2_shou`;

INSERT INTO users (username, password, avatar) VALUES
('admin', '123456', ''),
('test_user', '123456', '');
```

> 当前版本密码为**明文**存储，登录时直接输入 `123456` 即可。

## 六、配置后端数据库连接

打开项目根目录的 **`app.js`**，找到约第 35–43 行：

```javascript
const pool = mysql.createPool({
    host: 'localhost',
    user: 'root',
    password: '你的MySQL密码',   // ← 改成你自己的 root 密码
    database: '2_shou',
    waitForConnections: true,
    connectionLimit: 10,
    queueLimit: 0
});
```

| 配置项 | 说明 | 一般填什么 |
|--------|------|------------|
| host | 数据库地址 | `localhost` |
| user | 用户名 | `root` |
| password | 密码 | 安装 MySQL 时设置的密码 |
| database | 库名 | `2_shou`（不要改） |

保存文件。

## 十、可选配置

### 10.1 购物智能体（AI 助手）

默认使用内置规则回复，**无需配置**即可使用。

若要接入外部大模型 API：

1. 复制 `.env.example` 为 `.env`（项目根目录）：

```powershell
copy .env.example .env
```

2. 编辑 `.env`，填入真实地址和 Key：

```env

SHOPPING_AGENT_API_KEY=你的API密钥
```

3. 重启后端 `npm start`

> `.env` 已在 `.gitignore` 中，不会被提交到 Git。

### 10.2 修改前端 API 地址

若后端不在本机 3000 端口，编辑 `frontend/src/api/index.ts`：

```typescript
export const BASE_URL = 'http://localhost:3000';
```

开发环境下，Vite 代理配置在 `frontend/vite.config.ts`，一般无需修改。

## 十一、端口与防火墙

| 端口 | 程序 | 被占用时的表现 |
|------|------|----------------|
| 3000 | 后端 Express | `EADDRINUSE :::3000` |
| 5173 | 前端 Vite | 前端启动失败或换端口 |
| 3306 | MySQL | 后端报数据库连接失败 |

**查看 3000 端口占用（Windows）：**

```powershell
netstat -ano | findstr :3000
```

记下最后一列 PID，结束进程：

```powershell
taskkill /PID 进程号 /F
```

然后重新 `npm start`。

## 十二、常见问题排查

### Q1：登录按钮没反应 / 登录失败

| 可能原因 | 解决办法 |
|----------|----------|
| 后端未启动 | 确认 3000 端口有 `npm start` 在跑 |
| 数据库密码错误 | 检查 `app.js` 中 password |
| 用户不存在 | 先注册，或用 SQL 插入测试用户 |
| 密码输错 | 当前为明文，需与数据库中完全一致 |

---

### Q2：页面报 API 404（如 `/api/cart/2`、`/api/upload`）

| 可能原因 | 解决办法 |
|----------|----------|
| 后端是旧进程 | 关掉旧终端，重新 `npm start` |
| 只开了前端没开后端 | 必须先启动后端 |
| 改动了 app.js 未重启 | 后端不支持热重载，改完需重启 |

---

### Q3：`EADDRINUSE :::3000`

**原因：** 3000 端口已被占用（通常是之前启动的后端还在跑）。

**解决：** 不要重复执行 `node app.js`，或按「十一」结束占用进程后再启动。

---

### Q4：数据库连接失败 / `ECONNREFUSED 3306`

| 可能原因 | 解决办法 |
|----------|----------|
| MySQL 未启动 | 服务管理器中启动 MySQL，或用 Workbench 连接测试 |
| 密码错误 | 修改 `app.js` 中的 password |
| 库不存在 | 执行「5.1」创建 `2_shou` 数据库 |

---

### Q5：`npm install` 很慢或失败

```powershell
npm config set registry https://registry.npmmirror.com
npm install
```

---

### Q6：上传图片失败

| 可能原因 | 解决办法 |
|----------|----------|
| 后端未启动 | 启动 `npm start` |
| 文件超过 5MB | 换小一点的图片 |
| uploads 无写权限 | 确认项目根目录可写，后端会自动创建 `uploads/` |

---

### Q7：聊天消息不实时刷新

- 确认后端已启动（Socket.IO 与 Express 同端口 3000）
- 刷新页面后重试
- 两个浏览器窗口分别登录不同账号测试
