## 2. 系统架构图（System Architecture Diagram）

- [ ] Vite Dev Server（开发环境，端口 5173）

- [ ] Node.js 运行时

- [ ] MySQL 8 数据库（`2_shou`）

### 3.3 关系（Relationship）

| 关系 | 类型 | 说明 |

| users — orders（buyer） | 1 : N | 买家多个订单 |
| users — orders（seller） | 1 : N | 卖家多个订单 |
| orders — order_items | 1 : N | 一个订单多条明细 |
| users — messages（发送） | 1 : N | 用户发送多条消息 |
| users — messages（接收） | 1 : N | 用户接收多条消息 |
| users — addresses | 1 : N | 用户多个地址（可选） |

## 4. 流程图（Flowchart）

建议画 **3~4 张**，各选一条核心流程：

### 4.1 用户登录流程

- [ ] 开始
- [ ] 打开登录页
- [ ] 输入用户名、密码
- [ ] 表单校验（是否为空）
- [ ] 调用 POST /api/login
- [ ] 判断：账号密码是否正确？
  - 否 → 提示错误 → 结束
  - 是 → 保存用户信息到 localStorage
- [ ] 是否有 redirect 参数？
  - 是 → 跳转原页面
  - 否 → 跳转首页
- [ ] 结束

### 4.2 商品发布流程

- [ ] 开始（需已登录）
- [ ] 进入发布页
- [ ] 填写描述、价格
- [ ] 上传图片（可选，最多 9 张）
- [ ] 表单校验（描述、价格 > 0）
- [ ] 调用 POST /api/goods
- [ ] 判断：发布成功？
  - 否 → 提示错误
  - 是 → 跳转首页
- [ ] 结束

### 4.3 完整购物流程（重点）

- [ ] 开始
- [ ] 浏览/搜索商品
- [ ] 进入商品详情
- [ ] 选择：加入购物车 / 立即购买
- [ ] 进入结算页，填写收货信息
- [ ] 校验手机号、地址
- [ ] 调用 POST /api/orders 创建订单
- [ ] 按卖家拆单（多卖家 → 多个订单）
- [ ] 跳转订单详情
- [ ] 买家付款（POST /api/orders/:id/pay）
- [ ] 商品状态变为「已售」
- [ ] 卖家发货（POST /api/orders/:id/ship）
- [ ] 买家确认收货（POST /api/orders/:id/receive）
- [ ] 订单完成
- [ ] 结束

### 4.4 订单状态流转图（状态图风格）

- [ ] 待付款(0)
- [ ] → 付款 → 待发货(1)
- [ ] → 取消 → 已取消(4)（仅待付款可取消）
- [ ] 待发货(1) → 发货 → 待收货(2)
- [ ] 待收货(2) → 确认收货 → 已完成(3)

### 4.5 聊天消息流程（可选）

- [ ] 商品详情点击「联系卖家」
- [ ] 进入聊天页，加载历史消息
- [ ] 输入并发送消息
- [ ] 写入 messages 表
- [ ] Socket.IO 推送给对方
- [ ] 对方页面刷新消息列表

## 5. 时序图（Sequence Diagram）

建议画 **3~4 张**，标注参与者与消息顺序：

### 5.1 用户登录时序图

**参与者**

- [ ] 用户（Actor）
- [ ] LoginView（前端）
- [ ] Axios
- [ ] Express（/api/login）
- [ ] MySQL

**消息顺序**

1. 用户 → LoginView：输入账号密码，点击登录
2. LoginView → Axios：POST /api/login
3. Axios → Express：HTTP 请求
4. Express → MySQL：SELECT users WHERE username AND password
5. MySQL → Express：返回用户记录
6. Express → Axios：{ code:200, data: user }
7. LoginView → localStorage：保存 user
8. LoginView → 用户：跳转首页

### 5.2 加入购物车并下单时序图（重点）

**参与者**

- [ ] 买家
- [ ] GoodsDetailView / CartView / CheckoutView
- [ ] Pinia（cart store）
- [ ] Express API
- [ ] MySQL

**关键步骤**

- [ ] POST /api/cart（加购）
- [ ] GET /api/cart/:user_id（拉取购物车）
- [ ] POST /api/orders（创建订单，事务）
- [ ] DELETE cart_items（清购物车）
- [ ] 返回 order_ids

### 5.3 订单付款与发货时序图

**参与者**

- [ ] 买家 / 卖家
- [ ] OrderDetailView
- [ ] Express
- [ ] MySQL

**关键步骤**

- [ ] POST /api/orders/:id/pay → 更新 orders.status=1，goods.status=2
- [ ] POST /api/orders/:id/ship → 更新 orders.status=2，写入 shipping_no
- [ ] POST /api/orders/:id/receive → 更新 orders.status=3

### 5.4 实时聊天时序图

**参与者**

- [ ] 用户 A
- [ ] ChatView A
- [ ] Express + Socket.IO
- [ ] MySQL
- [ ] ChatView B（用户 B）
- [ ] 用户 B

**关键步骤**

- [ ] A 发送消息 → POST /api/messages
- [ ] Express 写入 messages 表
- [ ] Express → io.to(user_B).emit('new_message')
- [ ] ChatView B 收到事件 → GET /api/messages 刷新列表

### 5.5 购物助手时序图（可选）

**参与者**

- [ ] 用户
- [ ] ShoppingAgentView
- [ ] Express（/api/shopping-agent/chat）
- [ ] MySQL（查商品）
- [ ] 外部 LLM API（可选）
